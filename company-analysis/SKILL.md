## 核心功能

针对一个 F33-Micro 公司页面（或用户指定的上市公司），用 **K 线点段结构 + 横纵分析法**，将价格走势拆解为「节点（信息冲击）+ 段（影响发酵）」的交替序列，并把每个节点和每段与公司事件 / 行业逻辑对齐。

回答两个问题：

> 1. 这个**节点**为什么出现？什么信息冲击进入了市场？
> 2. 这一**段**为什么是涨/跌/震荡？上个节点的事件在如何发酵？

输出写入该 micro 页面的 **时间线** 段和 **盯什么** 段。

## 方法论基石

### 横纵分析法

> 纵向追时间深度，横向追同期广度，最后交汇出判断。

- **纵向**：沿时间线还原节点前后的完整故事——发生了什么、谁做的决策、为什么
- **横向**：在节点的同一时间窗口，对比同业竞品在做什么、市场在交易什么
- **交叉**：今天的价格形态，往往是更早一个不起眼的决策慢慢积累的结果

### 点段模型

- **节点 = 价格产生显著反应的区间**（几天到一两周）
  - 反应类型：反转 / 受阻 / 支撑 / 突破 / 多次测试后的最后一击
  - **不纠结具体是哪一天**，记反应窗口的起止区间即可
- **段 = 相邻节点之间的区间**，分类成 **涨 / 跌 / 震荡** 三类
- **段的本质 = 上一个节点事件的影响在被市场消化、发酵**
  - 涨段 = 基本面持续兑现 / 预期被持续上修
  - 跌段 = 预期被证伪 / 反向定价
  - 震荡段 = 多空对该事件解读分裂、信息不充分

### 模型自洽性

只有产生**显著价格反应**的事件才升格为新节点；没产生反应的信息会被段吸收（属于发酵的一部分）。因此 K 线 = 节点 + 段的完整交替序列，不会遗漏。

## 执行流程

### Step 1：读取目标页面

- 若用户指定了 micro 页面路径，直接 Read
- 若用户只说了公司名，用 `obsidian-cli` 搜索 `F33-Micro/公司与品牌/` 下匹配的页面
- 若页面不存在（新公司），新建一个最小骨架（时间线 / 盯什么 两个段）
- 记录已有 时间线 和 盯什么 内容，避免后续写入重复

### Step 2：拉取 K 线数据

用 `tradingview-quantitative` skill 或 WebFetch TradingView：

- **日线 OHLC**（近 1-3 年），叠加 MA20/MA60/MA120
- 周线视图由日线聚合而来（不需要单独拉一次）
- 标注成交量异常放大的日期作为辅助信号

若公司未上市或数据不可得，跳过 K 线相关步骤，仅做基本面事件梳理。

### Step 3：周线扫描——定大骨架

在**周线**视图上，标出所有**价格产生显著反应**的位置：

- 周期内的明显高点 / 低点（反转）
- 价格触及后回落或反弹的水平位（受阻 / 支撑）
- 被多次测试的价位（≥2 次触及形成的水平带）

输出：每个反应位的**价格区间**（一般是一个 range，不是单一价位）+ 反应类型。

**不限定数量**，但门槛是"肉眼可辨的显著反应"，长牛股可能 3 个、震荡股可能 6-8 个。

### Step 4：日线细化——确认节点区间

对 Step 3 给出的每个反应位，下钻到**日线**：

- 找到该价位附近反应发生的**时间窗口**（一般几天到一两周）
- 记录：**节点编号 | 反应区间起止日期 | 价格区间 | 反应类型**

例如：
```
N3 | 2025-08-12 ~ 2025-08-19 | 价格区间 48-52 元 | 反转（前期高点受阻回落）
```

### Step 5：切段打标签

把 K 线按节点切成段，相邻两个节点之间的区间标记为：

- **涨**：起止价差为明显正向，走势单调向上为主
- **跌**：起止价差为明显负向，走势单调向下为主
- **震荡**：价差在窄幅内来回，或有明确的箱体上下沿

由模型综合判断 + 输出判断依据（无硬阈值）。

输出形如：
```
N1 → 涨段 → N2 → 震荡段 → N3 → 跌段 → N4 → ...
```

### Step 6：节点纵向追溯——发生了什么信息冲击

对每个节点的反应区间，用 WebSearch 搜索：

```
<公司名> <反应区间日期前后1个月> 公告 财报 新闻 事件
```

重点查找：
- **财报发布**：营收/利润超预期或暴雷
- **重大合同/客户**：签新客户、丢老客户
- **产品发布**：新品上线、技术突破、量产消息
- **政策/监管**：行业政策变化、反垄断、补贴、关税
- **管理层变动**：CEO/CFO 更换、核心人员离职
- **融资/并购**：定增、配股、收购、被收购
- **行业事件**：竞品动态、上下游价格波动
- **宏观冲击**：利率、汇率、地缘事件

每找到一个事件，立即填入对应节点的"信息冲击"字段。**找不到事件**就标注 `[事件未匹配]`，不凭先验编造。

### Step 7：节点横向对比——α 还是 β

对每个节点的反应区间，搜索：

```
<竞品公司> <同一日期前后> 股价 新闻
```

回答：
- 竞品同期走势是否相似？
- 行业共振（同涨同跌）→ **β 驱动**
- 仅该股反应而竞品平稳 → **α 事件**（公司特有）

**横向对比必做**——至少找一个竞品对比，不能跳过。

### Step 8：段的发酵解释

对每一段，回答一个问题：

> **上一个节点的事件，在这一段时间里是怎么被市场继续消化/发酵的？为什么走出了涨/跌/震荡？**

形如：
```
N2 → N3 涨段（2025-04-15 ~ 2025-08-12，+34%）
  上个节点 N2 是"Q1 财报营收超预期 25%"
  发酵过程：卖方陆续上修盈利预测 → 北向资金持续流入 → 中报预告再次上修
  段内无新增节点级冲击，逻辑一致延续至 N3 受阻
```

或：

```
N5 → N6 震荡段（2025-09 ~ 2025-12，±8%）
  上个节点 N5 是"行业反垄断政策"
  发酵过程：多空对政策落地力度解读分裂，先跌后反弹再回落，
           等待 12 月细则出台 → 形成 N6 突破
```

### Step 9：交叉判断 + 写入页面

按下述 schema 写入目标 micro 页面：

#### 时间线段（按日期正序追加，先 grep 确认无重复）

```markdown
- **YYYY-MM-DD ~ YYYY-MM-DD [[公司名]] 节点 Nx（反转/受阻/支撑/突破）**：价格区间 X-Y
  - **信息冲击**：<纵向追溯出的事件>
  - **横向**：同期 [[竞品]] <走势>，判定为 <α / β / 混合>
  - [来源](url)

- **YYYY-MM-DD ~ YYYY-MM-DD [[公司名]] 段 Nx→Ny（涨/跌/震荡，±N%）**
  - **发酵逻辑**：<上个节点事件如何被消化>
  - **判断依据**：<为什么是涨/跌/震荡>
```

#### 盯什么段（覆盖更新，提炼 3-5 条）

```markdown
- <即将验证的事件或数字 1>
- <即将验证的事件或数字 2>
```

### Step 10：向用户输出摘要

```markdown
## <公司名> K 线复盘

### 点段序列
N1（反转）→ 涨段(+X%) → N2（受阻）→ 震荡段 → N3（突破）→ ...

### 节点 × 事件对照
| 节点 | 区间 | 反应类型 | 信息冲击 | α/β |
|---|---|---|---|---|
| N1 | ... | ... | ... | ... |

### 段 × 发酵对照
| 段 | 标签 | 上个节点的事件如何发酵 |
|---|---|---|
| N1→N2 | 涨 | ... |

### 核心发现
- 近 N 年主要驱动力是 <α/β/混合>
- N 个节点中，N 个是公司特有事件
- 当前处于 <最后一段的发酵阶段>
- 接下来盯 <1-2 个最关键变量>
```

## 约束

- **缺源不写**：找不到对应事件的节点，标注 `[事件未匹配]`，不凭先验知识编造
- **α/β 必须区分**：每个节点明确标注是行业共振还是个股独立
- **不覆盖已有内容**：时间线新增条目前先 grep 确认该日期区间无现有条目
- **横向对比必做**：至少找一个竞品做对比
- **节点是区间不是点**：记反应窗口的起止日期，不要硬选某一天
- **段必须给发酵解释**：不能只贴标签，要回答"上个节点的事件如何被消化"
- **只写事实层**：写"发生了什么 + 市场如何消化"，不做估值判断和买卖建议
