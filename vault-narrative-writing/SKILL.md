# Narrative 写作方法

## 概述

编写 F31-Macro/*/Narrative/ 目录下的叙事页面。每个 narrative 讲一个历史事件的故事——它的背景、经过和留下的影响。

核心原则：**每篇 narrative 应该是一条因果链，而不是一个事实列表。**

## 结构模板

每篇 narrative 必须包含以下六个部分，按顺序组织：

### 1. 一句话主线（blockquote）

用一个 blockquote 概括**事件的本质 + 最关键的结果/判断。**

- 2~3 句话以内
- 必须有判断，不能只有描述
- 代表当前认知版本

示例：
- ✅ "一场冷战格局下的联二打一——中国和美国都在寻找第三种力量改变美苏两极对峙的僵局。这次握手改变了后来的世界走向。"
- ✅ "中国债务周期中'居民接过最后一棒'的政策操作。以央行 PSL 印钱→国开行放贷→拆迁户买房为链条，实现了去库存、救财政、拉房价的三重目标。"
- ❌ "1972 年 2 月 21-28 日尼克松访华，终结两国 25 年相互隔绝"（只有事实，没有判断）

### 2. 为什么会发生

回答"为什么会有这件事？"

如果涉及多方博弈，从每一方的视角分别解释其动机。

使用平实的语言。用问题作为段落标题。

### 3. 事件本身

解释核心机制/关键决定/转折节点。

使用 **比喻和类比** 解释复杂机制。示例：
- "吸星大法"——中国引资策略：跟对手抱在一起吸取内力
- "联二打一"——跟次要敌人结盟对抗首要敌人
- "涨价去库存"——通过推高房价消化库存

### 4. 连锁反应

用 **阶段演进** 组织，不是一次性列举所有结果：

- 每阶段有清晰标题+时间区间
- 每阶段要解释：这个阶段的特征 → 为什么进入这个阶段 → 导致了什么
- 阶段之间必须因果衔接：因为阶段 1 → 所以阶段 2

### 5. 留下了什么

回答"为什么今天还要了解这件事？"

从多个维度覆盖（对每个参与者、对整体格局、对后续事件），每个维度 1~2 句话。

### 6. 主线演化

页底日志格式。只在核心判断翻转或重写时追加。

```
## 主线演化

- YYYY-MM-DD: 初始版本——[一句话判断]
```

## 写作规则

### 规则 1：用问题驱动段落

每个段落标题应该是读者会自然提出的问题：
- "为什么会走到这一步？"
- "货币化棚改是什么？"
- "谁在承担代价？"
- "留下了什么？"

即使读者还不知道答案，也能被问题带着往下走。

### 规则 2：因果链，不堆事实

❌ 平铺时间线：
```
- 1950-06-25 朝鲜战争爆发
- 1950-09-15 仁川登陆
- 1950-10-25 中国参战
```

✅ 因果叙事：
```
第一阶段朝鲜军势如破竹 → 第二阶段仁川登陆逆转 → 第三阶段中国参战再次逆转 → 第四阶段僵持谈判
```

每个转折必须解释：**因为之前发生了什么，所以接下来发生什么。**

### 规则 3：数据服务于故事

- 只保留 1~2 个最核心的数据（如"储备从 4 万亿降至 3 万亿"、"库存 7.18 亿平方米"）
- 表格只用于对比（如"三轮加杠杆的差异：企业 vs 政府 vs 居民"）
- 不要把数据表格作为主体结构

### 规则 4：双视角叙述

涉及多方博弈的事件，从各方视角解释动机：
- 尼克松访华：中国"联美抗苏" vs 美国"打中国牌"
- 朝鲜战争：朝鲜（武力统一） vs 美国（遏制共产主） vs 中国（边境安全）
- 811 汇改：汇率市场化需求 vs 入篮 SDR 要求

### 规则 5：关键节点要有判断

- 在每个阶段/转折点给出明确判断
- 不要用"据分析""有专家认为""市场普遍认为"这类模糊主语
- blockquote 本身就是判断——写得有底气

### 规则 6：用平实的语言

- 首次出现的术语要解释
- 用短句
- 用具体语言代替抽象表述
- 把政策/官方语言翻译成日常语言

## Frontmatter 要求

每篇 narrative 必须包含：

```yaml
---
title: YYYYMM 事件名称（如 201508 811汇改）
tags: [type/narrative/regional 或 type/narrative/country, ...]
links:
  - "[[关联 Entity 1]]"
  - "[[关联 Entity 2]]"
start_date: YYYY-MM-DD
end_date: YYYY-MM-DD
date created: YYYY-MM-DDTHH:MM
date modified: YYYY-MM-DD
---
```

- `start_date` 和 `end_date` **必填**
- `end_date` 如果确切结束时间不明确，可以填估算日期

## 禁止事项

1. 不要用"背景→内容→意义→影响"的教科书结构
2. 不要用要点列表式时间线作为主体结构（时间线应该嵌入因果阶段内部）
3. 不要堆砌数据（数据是证据，不是叙事本身）
4. 不要写孤立段落（每段必须与前后因果衔接）
5. 不要用模糊来源（"有分析指出""市场认为"）——用具体来源
