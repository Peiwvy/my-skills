## 整理规则

### 一、核心原则

> [!important] 信息完整性 > 一切风格指标
> 讲座中每句话默认保留，只删：口语填充词（嗯/啊/那个）、完全重复的表述、无关寒暄。
> 宁可多写、不要漏信息——但**禁止为凑长度灌水**。

| 必须保留 | 处理方式 |
|---------|---------|
| 核心论点 | `> [!quote]` 保留原话 |
| 因果机理 | 完整保留逻辑链条，不跳步 |
| 硬数据（数字/年份/百分比/金额/人名） | 全保留，用 `**加粗**` |
| 生动比喻 | `> [!example]` 原汁原味 |
| 真实案例 | 完整保留过程 + 关键数字 |
| 常见误区 | `> [!warning]` 标记 + 说明"为什么错" |
| 实操建议 | `> [!tip]` 标记 |
| 讲者提到的图表 | 插入图表 placeholder |
| **原文中的图片链接** | ==原样保留 `![](url)` Markdown 图片==，按原文位置插入到对应章节 |
| **讲者口头禅/反问/吐槽/自嘲** | ==原话保留==，用 `> [!quote]` 或正文引号 |
| **讲者第一人称判断** | 保留"我建议"、"我说"、"放心"、"我是没法回答"等口吻 |

#### 精炼方式

- ✅ 去口语填充词、合并重复表述、用更简洁的语言表达同样意思
- ✅ ==保留讲者特色语言==：反问、吐槽、自嘲、口头禅、"我说……"、"放心……"等
- ❌ 删除展开说明、跳过推理步骤、把生动比喻改成正式语言
- ❌ ==把讲者第一人称视角改成第三人称客观陈述==（教材化）

**示例 1：去填充词**
> 原文："那么，嗯，我们可以看到，其实，这个问题呢，它背后的逻辑是什么呢？就是说，当经济下行的时候，对吧，企业的盈利会下降，然后呢，股价就会跌。"
>
> 精炼后：背后的逻辑是：经济下行 → 企业盈利下降 → 股价下跌。

**示例 2：保留讲者口吻**
> 原文："你怎么回答？我是没法回答。这算什么话。"
>
> ❌ 错（教材化）：这种解释缺乏严谨性。
>
> ✅ 对（保留个性）：> [!quote] 付鹏吐槽
> > "==你怎么回答？我是没法回答==。这算什么话。"

#### 讲者风格保留专项

> [!important] 为什么要保留个性
> 讲者的反问、吐槽、自嘲、激将不是"修辞"，而是==帮助读者建立直觉、留下印象==的关键。
> 抹掉个性 = 总结变成"标准教材" → 读完没印象、抓不住重点。

##### 必保留的讲者风格元素

| 类型 | 例子 | 保留方式 |
|------|------|---------|
| **反问句** | "这怎么做交易？" "你怎么回答？" | 原句保留在正文或 quote |
| **吐槽/激将** | "这算什么话" "管它什么先干再说" | quote callout |
| **自嘲/谦逊** | "其实赚钱都不容易" "我也不知道" | 正文引号保留 |
| **口头禅** | "放心" "还是那句话" "我说" | 原样保留，不要"翻译" |
| **第一人称建议** | "我建议不要这样" "我很少去……" | 保留"我"，不要改成"应该" |
| **个人化判断词** | "好做" "没法做" "败了" | 保留口语化判断，不要换成"较容易/不可行" |
| **直接对读者喊话** | "大家什么时候见过……" "你们会发现……" | 保留"你/大家"，不要去人称化 |

##### 风格保留 vs 信息精炼的平衡

==**信息精炼的对象是"啰嗦的填充词"，不是"讲者的味道"**==。

- 同一观点重复 3 遍 → **合并到 1 遍**（精炼）
- 但合并后那 1 遍 → ==用讲者最有味道的那个版本==（保留风格）

**判断标准**：删掉这句话之后，**读者还能感受到"是这个人在讲"吗**？
- 不能 → 这句是讲者的风格锚点，必须保留
- 能 → 可以精炼

##### ⚠️ 风格保留的"度"——避免堆砌

> [!warning] 风格保留 ≠ 大段抄原话
> 保留风格不是"把所有讲者原话都摆进来"——堆砌反而稀释了风格的冲击力，让总结变冗余。
>
> ==**每个论点最多 1 个 quote callout**==，挑最有味道的一句保留，其余融入正文叙述（用动词/短引号"嵌入"）。

| 反模式 | 修正方式 |
|--------|---------|
| 同一论点连续 3 个 quote callout | 合并为 1 个 quote + 正文承接 |
| 每个小节都有 2 个以上 quote | 全篇 quote 总数控制在 ==每千字 ≤ 3 个== |
| Quote 内文本 > 5 行 | 拆成"短引号金句 + 正文展开" |
| 整段抄原文当 quote | 摘最锐利的一两句即可 |

**正文嵌入式保留风格的写法**：
- ❌ 堆砌：> [!quote] 付鹏吐槽 "==这怎么做交易==？让他持有 3 年……"
- ✅ 嵌入：付鹏对此直接吐槽："==这怎么做交易==？"——他认为问题不在框架，而在期望太高。

==**口头禅、反问、吐槽用短引号嵌入正文，金句和反直觉判断才用 quote callout**==。

### 二、Obsidian 格式

#### Frontmatter

```yaml
---
title: 讲者姓名-主题关键词-总结
tags: [state/source/sorted]
date: YYYY-MM-DD
up:
  - "[[原课程/源文件链接]]"
url: 原始链接（如有）
---
```

#### 标题层级

- `## 一、二、三` 一级大主题
- `### 1.1` 子节（仅在原文有明显逻辑断点时使用）
- 大主题之间用 `---` 分隔

#### Callout 类型与使用边界

> [!warning] 节制原则
> ==Callout 用来"强调"，不是"装饰"==。平铺叙述、过渡、解释性内容 → 用**普通段落**。每段都包 callout = 强调失效。

| Callout | 使用场景 | 频率上限 |
|---------|---------|---------|
| `> [!quote]` | 讲者原话**金句**（不是普通陈述）| 每章节 1-2 处 |
| `> [!example]` | 生动比喻、案例故事 | 按需 |
| `> [!tip]` | 实操建议 | 按需 |
| `> [!warning]` | 常见误区、反直觉提醒 | 按需 |
| `> [!important]` | 全篇最关键的结论 | 每篇 ≤ 3 处 |
| `> [!note]` | 图表 placeholder、补充说明 | 按需 |
| `> [!abstract]` | 仅用于结尾总结 | 每篇 1 处 |
| ~~`[!info]`~~ | ❌ 不用作"本节定位"等装饰——改用首段普通文字 | — |

**判断标准**：删掉 callout 视觉框只看文字，如果这段是文章中"信息密度最高、最值得注意"的 10-20% → 用 callout；否则用普通段落。

#### 强调标记

##### `==高亮==` 严格限定三类语义

| # | 场景 | 示例 |
|---|------|------|
| 1 | 讲者原话的洞察性金句 | "==人生发财靠康波==" |
| 2 | 反直觉结论（多数人会想错的判断）| "==老龄化反而会推高汇率==" |
| 3 | 全文最关键的数据/数字（不是所有数据）| "==2008 年雷曼破产时杠杆率达到 33 倍==" |

每个小节 1-2 处即可，**不要每段都标**。普通强调用 `**加粗**`：关键概念、专有名词、一般数据/年份/百分比。

##### 高亮 vs 加粗的判断

> 这句话**反直觉**吗？是**金句**吗？是**全文最重要的数据**吗？
> - 是 → ==高亮==
> - 否 → **加粗**（甚至不强调）

#### 表格优先

==出现以下结构信号 → 强制用表格==：

| 信号 | 例子 |
|------|------|
| 重复 ≥3 次的并列项 | "第一……第二……第三……" |
| 多对象对比 | 前/后、正/误、A 国 vs B 国 |
| 时间线 | 年份 + 事件 |
| 分类清单 | "可分为以下几类" |
| 多维度评分 | 一个对象多属性、多对象同属性 |

判断标准：列表项 ≥3 且每项有相同子结构 → 改表格。

#### 标题划分

- 子节 `###` 划分依据：==原文有明显逻辑断点==（讲者主动说"我们再看下一个问题"、"接下来谈"）
- ❌ 不为凑结构对称硬拆 1.1 / 1.2 / 1.3
- ❌ 不让一个子节内容只有一两句话——合并到上一节
- ❌ ==一个 `##` 下只有一个 `###` → 去掉 `###`==，内容直接挂在 `##` 下（独苗子节没有划分意义）

#### abstract 总结块

==**abstract = 全文思路 + 结论性话语**==——像读完一本书后给朋友讲"这本书在说什么"。

✅ Good 示例：
> [!abstract] 总结
> 戴康从债务周期视角看资产配置：在逆全球化 + 全球央行 2022 起去美元化的背景下，黄金从过去"涨幅落后"的资产升级为**未来多年最值得配置的核心资产**——本质价值不在于美债利率共振或避险溢价，而在于**超国家主权信用**。同时面对中国"内卷=债务周期"的现实，企业唯一的破局之道是出海……

❌ Bad（禁用流水账清单）：
> [!abstract] 总结
> 1. 戴康反直觉：黄金是最看好的资产
> 2. 多数人思维定式：黄金不是最佳投资选择
> 3. ……（30+ 条）

要求：
- ❌ 不列数字 takeaway / 不重排正文要点 / 不写 20+ 条流水账
- ✅ 写思路（A → B → C 论证主线）+ 结论（讲者最终想让你记住的判断）

#### 图表 Placeholder

讲者提到图表时插入：
```markdown
> [!note] 📊 图表 Placeholder
> **原讲座此处有图表**：[描述]
>
> 图表要点：
> - [要点1]
> - [要点2]
```

### 三、验证步骤（强制不可跳过）

#### 第 0 步：读取完整性预检（开始写总结前必须执行）

read_file 默认最多显示 200 行，原文可能 300-500+ 行。**未读完就写总结 → 必然漏后半部分**。

强制流程：
1. 读取时记录 `full_length: N`
2. 200 行以内 1 次读完；201-400 行至少 2 次；401-600 行至少 3 次……
3. 开始写之前确认：**已读行数 = full_length**，否则继续读
4. 工具不会报错，只会截断 → `full_length` 是唯一标准

#### 五项检查（逐项执行）

对比原文与总结，识别缺失，立即补充。

| 检查项 | 检查内容 | 缺失时 |
|--------|---------|-------|
| **1. 逐段回读** | 每个"为什么"是否有完整解释；每个结论是否有推导过程 | 只有结论无机制 → 回原文补 |
| **2. 逻辑链** | 所有"A 导致 B"是否解释了"如何导致"；中间传导步骤是否保留 | 跳步 → 补完整链条 |
| **3. 展开说明** | 核心概念是否保留讲者展开解释和"为什么这样"的原理 | 只有定义无解释 → 补 |
| **4. 案例细节** | 案例是否保留关键数字、步骤、完整过程 | 只有结论无过程 → 补 |
| **5. 反直觉逻辑** | 反常识结论是否详细解释"为什么"，有具体计算或推导 | 解释不足 → 补原理 |

#### 常见缺失类型（警惕）

- **机制缺失**：只写"A→B"不写"如何导致" → 搜索"→"符号检查
- **步骤缺失**：只写起点终点，跳过中间过程
- **原理缺失**：只写操作不写为什么这样操作
- **计算缺失**：只写结果不写计算过程
- **风险缺失**：只写操作不写风险

**自查终极方法**：删掉某段话，读者还能理解后续吗？
- 不能 → 必须保留
- 能 → 可能是冗余口语填充

### 四、典型错误对照（4 个高频反例）

#### ❌ 错误 1：删除展开说明

> 原文："为什么社融指标失效了？过去社融主要流向房地产和基建，带动经济，股市自然上涨。但现在社融变了，资金沉淀在银行体系，没进入实体经济，所以社融增速高，股市却不涨。"
>
> ❌ 错：社融指标失效了，因为社融流向变了。
>
> ✅ 对：
> ==社融与股市的关系 2022 年后断裂==。
> - **过去**：社融 → 房地产/基建 → 经济 → 股市上涨
> - **现在**：社融 → 银行体系沉淀 → 没进实体 → 社融高股市不涨

#### ❌ 错误 2：跳过逻辑链

> 原文："老龄化导致汇率走强。因为消费萎缩，但产能不会立刻下降，产能过剩国会增加出口，贸易顺差推高汇率。"
>
> ❌ 错：老龄化导致汇率走强。
>
> ✅ 对：==老龄化会导致汇率走强==，传导链条：
> 老龄化 → 消费萎缩 → 产能过剩 → 增加出口 → 贸易顺差 → 汇率走强
> 关键点：**产能调整滞后于需求变化** → 供需错配。

#### ❌ 错误 3：只写结果不写计算

> 原文："美国企业到欧洲发欧元债，通过货币基差互换。支付美元 LIBOR，收到欧元 LIBOR + 基差。基差是负数（如-150bp），减去负值，欧元 LIBOR 接近 0，结果收到的利息反而比支付的多。"
>
> ❌ 错：利用欧美利差+负基差双重套利。
>
> ✅ 对：**为什么能"借钱反而收钱"**：
> - 支付：美元 LIBOR
> - 收到：欧元 LIBOR + 基差（负数 -150bp）
> - 因基差是负数，实际是**减去负值** → 欧元 LIBOR -(-150bp) = 欧元 LIBOR + 150bp
> - 欧元 LIBOR 接近 0 或负利率 → 收到 > 支付

#### ❌ 错误 4：抹掉讲者个性（教材化）

> 原文（付鹏）："放心，两三年这样的周期的行情，你也只是看得非常清楚，就算 500 倍杠杆、400 倍杠杆、200 倍杠杆进去，这个钱你也赚不着的，因为没法进行策略的设计和风控的管理。换句话说，到这个环节上你就已经败了，前面看得再对也没用。"
>
> ❌ 错（教材化、无人味）：
> 即使长期方向判断正确，高杠杆下仍难以获利，因为策略和风控难以实施。
>
> ✅ 对（保留付鹏吐槽口吻）：
> > [!quote] 付鹏激将
> > "==放心==，两三年这样的周期的行情，你也只是看得非常清楚，就算 **500 倍、400 倍、200 倍杠杆**进去，==这个钱你也赚不着的==——因为没法进行策略设计和风控管理。换句话说，==到这个环节上你就已经败了，前面看得再对也没用==。"

判断：原文里"放心"、"你也赚不着"、"已经败了"——这些是付鹏的招牌语气，==删了就是另一个人在讲==。

#### ❌ 错误 5：丢图表说明

> 原文："横轴时间纵轴三类资产相对表现。2020 景气成长，2021 经济周期，2022-2023 稳定价值，2024 又回到景气成长。"
>
> ❌ 错：三类资产在不同时期有不同表现。
>
> ✅ 对：插入图表 placeholder + 完整时间线表（2020/2021/2022-2023/2024 各自占优主线）。
